软件开发版本控制

九月 17, 2020

在软件开发过程中主要分以下几个版本历程

版本 环境 说明
developer alpha 最基本的开发环境
release_xxx beta 预发布的新版本
staging staging 模拟线上测试
master mater 灰度测试
vxxx.xxx(tag) pro 生产环境
fix-xxx 修复bug
feature/xxx 新特性的分支

版本迭代历程
developer -> release_xxx -> staging -> master -> vxxx.xxx(tag)

结合 Gitlab 的ci，可以便捷的控制代码的分支对应不同的环境，实现代码提交后自动编译发布到对应的环境
